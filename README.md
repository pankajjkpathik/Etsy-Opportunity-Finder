# Etsy-Opportunity-Finder
Etsy Opportunity Finder
Etsy Opportunity Finder — Solo-Founder MVP Blueprint

A realistic, ship-in-90-days plan. Every module is scoped to what a solo founder can actually build with Next.js + Firebase/Supabase + a Chrome Extension, using only observable Etsy signals (public listing pages, public shop pages, public search results). No invented metrics, no fantasy data lakes.

One honest constraint up front: Etsy doesn't expose search volume or sales data, and their ToS restricts large-scale scraping. The architecture below leans on the official Etsy Open API v3 for structured data, user-initiated client-side fetches via the Chrome Extension for page-context analysis (legally cleaner than server scraping), and OpenAI/Gemini for synthesis. Anywhere a number is an estimate, the UI shows the inputs and a confidence band.

Feature Breakdown
Module 1 — Product Opportunity Finder (flagship)

Inputs: keyword or niche string.

What we fetch (Etsy Open API v3 findAllListingActive + getListing):

• Listing count for the query
• For top 100 listings: price, creation date, numfavorers, shopid, views (when exposed), tags
• For unique shops in result: shop creation date, total review count, total listing count

Opportunity Score (0–100) — transparent formula:

`
opportunityscore =
    0.30  competitioninverse      // 100 - normalized(listingcount, nichebaseline)
  + 0.20  reviewdensity            // log(medianreviewspertoplisting) scaled 0-100
  + 0.15  pricestrength            // medianprice vs. category median, capped
  + 0.15  newlistingvelocity      // % of top-100 created in last 90 days
  + 0.10  favoritevelocity         // median favorites / listing age in days
  + 0.10  shopdiversity            // 100 - HHI of top-100 by shopid
`

Every term is shown in a breakdown panel. Hovering each shows its raw inputs. If any input is missing, the term is dropped and the weights are renormalized — and a "partial data" badge appears.

Output card:
• Score + risk level (Low/Medium/High based on competition + saturation)
• Plain-English explanation generated by AI from the numeric breakdown
• Launch recommendation: "Test with 3–5 variants" / "Crowded — only enter with differentiation" / "Pass"
• Confidence: % of formula terms that had real data

Module 2 — Niche Finder

Same data pipeline as Module 1, but aggregated. Input is a category/subcategory or keyword stem. Returns:

• Total active listings
• Median + IQR pricing
• Reviews-per-listing distribution
• Shop concentration (Herfindahl index → "dominated by 3 shops" vs. "fragmented")
• Maturity flag: % of listings older than 2 years
• Alert badge: Low / Moderate / Highly Saturated based on listingcount thresholds and shop concentration

Module 3 — Trend Momentum Engine

No prediction. Only observed deltas. We snapshot the top 100 listings for each tracked keyword nightly (Vercel cron). Over time we build:

• New listings added per week
• Aggregate review delta
• Aggregate favorite delta
• Median price drift

Momentum Score = z-score blend of 7d/30d/90d growth rates, normalized per niche. Direction labels (Rising/Stable/Declining) come from sign + magnitude. Sparkline + raw counts always visible. Confidence = number of snapshots available (need ≥4 for "high").

For brand-new keywords, we show "Tracking started — momentum available in 7 days" rather than fabricating a number.

Module 4 — Competitor Analyzer

Input: Etsy shop URL or handle. Pulls from getShop + findAllShopListingsActive:

• Shop age, total reviews, active listings
• Price range and median per category they sell in
• Tag frequency map → identifies their top keyword themes
• Listing-age distribution → "how often they ship new products"

Gap analysis (AI layer):
• Categories they sell in but underweight (few listings)
• Keyword themes present in successful competitors but missing from this shop
• Price tiers they don't occupy

We do not estimate revenue. We say "review velocity suggests ~X reviews/month" with the formula shown, and explicitly note that reviews ≠ sales (Etsy review rate ~20–40% per public seller reports — this is shown as an assumption the user can adjust).

Module 5 — Keyword Sniper

Pure client-side utility (no API cost):

• Paste/import list → dedupe, normalize case, strip emojis
• Sort by length, alpha, char count (Etsy 20-char tag limit highlighting)
• Cluster via embedding similarity (one Gemini embedding call per batch, cached)
• Expand: long-tail (suffix patterns), buyer-intent ("personalized", "custom", "for [recipient]"), question keywords
• Export CSV / XLSX / TXT

Module 6 — AI Listing Generator

Single template, multiple outputs from one generation pass to save tokens:

• Inputs: keyword, product type, audience, tone, optional unique angle
• Outputs: title (140-char Etsy limit), description (with hook → features → care → shipping structure), 13 tags (≤20 chars each, validated), FAQ block, alt text, Pinterest title/description, IG/FB caption, email blurb
• SEO check: keyword density, tag uniqueness, title front-loading score

Module 7 — POD Profit Calculator

Pure JS, no backend. Inputs as specified. Outputs include break-even, minimum viable price (target margin slider), max discount before unprofitable. Warnings: margin <15%, ad cost >25% of price, break-even above category median.

Localized: INR/USD toggle, GST field for India users.

API Structure

REST under /api/v1/*, Next.js Route Handlers. JWT (Firebase ID token) in Authorization: Bearer. Zod validation on every input.

| Method | Path | Purpose |
|---|---|---|
| POST | /auth/session | Exchange Firebase token for app session cookie |
| GET | /me | Current user + plan + usage counters |
| POST | /opportunities/analyze | Run opportunity finder for a query |
| GET | /opportunities | List saved opportunities |
| GET | /opportunities/:id | Detail + full breakdown |
| POST | /niches/analyze | Niche-level aggregation |
| POST | /trends/track | Start tracking a query |
| GET | /trends/:query | Historical snapshots + momentum |
| POST | /competitors/analyze | Shop analysis |
| POST | /keywords/process | Dedupe/cluster/expand |
| POST | /listings/generate | AI listing generator |
| POST | /profit/calculate | Stateless calc (also runs client-side) |
| POST | /billing/checkout | Stripe checkout session |
| POST | /billing/portal | Stripe customer portal |
| POST | /webhooks/stripe | Plan sync |
| POST | /extension/context | Extension pushes page context for analysis |

Cross-cutting concerns:
• Rate limiting: Upstash Redis sliding window — free 20/day, starter 200/day, pro 2000/day on heavy endpoints (opportunity/niche/competitor/listing). Soft warnings at 80%.
• Caching: Etsy API responses cached 6h by query hash; AI listing outputs cached by hash(inputs) for 24h.
• Errors: RFC 7807 problem+json shape.
• Idempotency: Idempotency-Key header honored on POSTs that cost money/quota.
